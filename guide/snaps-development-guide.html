# Resources

To get started with Snaps, you first have to download MetaMask Flask (opens new window).

While this guide is under construction, we highly recommend consulting the readme of @metamask/snaps-cli (opens new window), the command line utility we created to help with snap development.

In addition, you can review the growing number of example snaps (opens new window) maintained by MetaMask.

Finally, you may wish to check out the first functional key management snap ever made, the Filecoin Snap, or filsnap (opens new window).

# Gotchas

# Accessing the Internet in a Snap

Snaps do not get access to any sensitive APIs or features by default, and Internet access is no exception to that rule. To access the Internet, you must be specify the permission endowment:network-access in the initialPermissions of your snap.manifest.json file. This will grant you access to the global fetch API. Other global network APIs may be made available in the future.

# Writing SES-Compatible JavaScript

Snaps is implemented using Secure EcmaScript, or SES (opens new window), effectively a subset of the JavaScript language designed to enable mutually suspicious programs to execute in the same JavaScript process (or more accurately, the same realm (opens new window)).

For present purposes, the important thing to understand is that, because SES is effectively a subset of JavaScript, you may write - or more likely, import - code that does not execute under SES. @metamask/snaps-cli (opens new window) will attempt to fix some such issues for you via the mm-snap build command, but it can't fix them all.

When mm-snap build fails during the eval step (you can also directly execute a built snap using mm-snap eval), you have to attempt to fix the issue manually. If the problem is in a dependency, you can try a different version or make use of tools such as patch-package (opens new window) to fix the issue locally yourself.

If you're still stuck, you can ask for help here (opens new window).
